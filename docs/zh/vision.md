# 愿景

**Sonic Atlas** 的主线是：

> 通过机器学习（Machine Learning，用数据驱动模型自动学习规律的方法）方法，从音乐与自然声音中提取抽象声音特征，并将这些特征作为可听、可操作、可组合的音乐创作素材。

## 动机

传统音频分析通常服务于分类、检索、压缩或重建；Sonic Atlas 关注更进一步的问题：

1. 声音可以被分解成哪些层次？
2. 机器学习能否从声音中提取抽象但稳定的特征（Feature，对原始数据的数值化描述）？
3. 这些特征是否对应人耳可闻、可描述的变化？
4. 这些特征能否成为作曲、音色设计（Sound Design，通过合成与处理塑造声音的实践）和声音想象的材料？

## 核心管线

```text
Audio（原始音频）
  → Decomposition（分解）
  → Abstract Feature Extraction（抽象特征提取）
  → Perceptual Validation（感知验证）
  → Creative Mapping（创作映射）
  → Composition System（作曲系统）
```

## 设计原则

- **感知优先于指标**：特征不仅要在模型中有效，也必须在人耳中有意义。
- **控制优先于新奇**：生成新声音不够，关键是能否控制变化方向。
- **混合表征**（Mixed Representation）：不追求单一万能潜变量（Latent，模型学到的隐藏低维表示），而采用心理声学特征、稀疏原子（Sparse Atom，通过稀疏分解从信号中提取的时频基元）、深度嵌入（Embedding，将高维数据映射到低维连续向量的技术）、生成式潜变量/标记（Token，离散化的最小表示单元）的组合。
- **创作即评估**：一个特征最终是否有价值，要看它能否支撑声音作品或作曲过程。

## 研究主张

声音特征可以分为四层材料：

| 层级 | 单位 | 创作角色 |
|---|---|---|
| 心理声学描述符 | 标量 / 曲线 | 可命名控制参数 |
| 稀疏原子/模板 | 事件 / 原子 / 激活 | 声音粒子与素材重组 |
| 嵌入（Embedding） | 向量 / 邻域 | 语义导航与素材检索 |
| 生成式潜变量/标记（Token） | 潜变量 / 离散编码 | 变形、重合成（Resynthesis，基于特征重新生成音频）、生成 |

Sonic Atlas 的目标是把这四层统一到一个可实验、可听、可创作的系统中。
